## 2. 요약 (Executive Summary)

플로우보드는 엔터프라이즈 팀 리더가 팀원의 업무 부하, 일정 준수율, 협업 활동을 한눈에 파악할 수 있는 **팀 성과 대시보드**를 2030년 Q3에 출시한다. 현재 리더 사용자는 주간 보고서를 수작업으로 취합하는 데 평균 4.2시간을 쓰고 있으며, NPS 응답자의 38%가 "팀 현황 가시성 부족"을 1순위 불만으로 지적했다. 본 기능은 팀 리더의 정보 취합 시간을 75% 단축하고, 엔터프라이즈 고객의 유료 플랜 전환율을 12%p 끌어올리는 것을 목표로 한다. 타깃 사용자는 50인 이상 조직의 **팀 리더(1차)** 와 **팀원(2차)** 이며, 데이터 기반 1:1 코칭과 공정한 리소스 재배분을 즉각 지원한다.

## 3. 배경 및 문제 정의

### 3.1 사용자 리서치 및 고객 지원 데이터 요약

- **사용자 인터뷰(2030-02-10 ~ 2030-03-15, n=27)**: 팀 리더 27명 중 22명(81%)이 "팀원의 업무 과부하를 제때 인지하지 못한다"고 응답.
- **CS 티켓 분석(2029-10 ~ 2030-03)**: "리포트/대시보드 관련" 문의가 총 1,842건으로 전체 기능 문의의 **19%** 를 차지. 이 중 **64%** 가 "팀 단위 요약 뷰 부재"와 연결됨.
- **NPS 설문(n=612)**: Detractor 그룹(38%)이 "팀 단위 가시성 부족"을 Top 1 불만으로 지목, "보고서 자동화"가 Top 3.
- **사용 로그 분석**: 엔터프라이즈 플랜 리더 사용자의 평균 **주간 4.2시간**이 수동 보고서 작성에 소요되는 것으로 추정(프로젝트 뷰 체류 시간·스프레드시트 내보내기 빈도 기반).
- **이탈 인터뷰(n=9)**: 이탈 고객의 6/9가 "경쟁사의 대시보드 기능이 더 강력해서" 전환했다고 응답.

### 3.2 경쟁사 벤치마킹 요약

| 제품 | 팀 부하 가시화 | 일정 준수율 | 협업 활동 분석 | 실시간성 | 커스텀 위젯 |
|------|---------------|-------------|----------------|----------|-------------|
| 플랜웍스 Insights | 부분 지원 | 지원 | 미지원 | 15분 지연 | 5종 |

### 3.3 해결하려는 핵심 문제 3가지

1. **가시성 부재**: 리더가 팀원별 업무 부하·지연 리스크를 실시간으로 파악할 수 있는 통합 뷰가 없다.
2. **보고서 수작업 부담**: 주간 보고서 취합에 평균 4.2시간이 소요되어 리더의 전략 업무 시간이 잠식된다.
3. **리소스 재분배 근거 부족**: 과부하·저부하 상태를 정량적으로 증빙할 데이터가 없어 공정한 업무 재분배가 어렵다.

## 4. 목표 및 성공 지표

### 4.1 비즈니스 목표

1. 엔터프라이즈 플랜 **전환율 +12%p**(현 28% → 목표 40%, GA 후 6개월 시점).
2. 팀 리더 세그먼트의 **월간 이탈률 -30%**(현 2.4% → 목표 1.7%).
3. 확장 계약(Expansion ARR) **+USD 1.8M**(FY2030 하반기).

### 4.2 사용자 목표

1. 팀 현황 파악에 쓰는 시간을 **주당 4.2시간 → 1시간 이하**로 단축.
2. 팀원 업무 부하 이상 신호를 **48시간 이내** 감지.
3. 1:1/리뷰 미팅에 사용할 **데이터 기반 근거**를 3클릭 이내 확보.

## 5. 페르소나

### 5.1 페르소나 A — 팀 리더 "강민수" (38세, PMO 팀장)

- **소속**: 국내 대형 이커머스사, 사내 PMO 팀 12명 리드
- **페인포인트**
  - 월요일 아침 임원 리포트 작성을 위해 팀원별 진행 상태를 수동 취합.
  - 팀원 A의 번아웃 징후를 늦게 인지해 스프린트 2회 연속 지연.
  - 재택/사무실 혼합 근무 환경에서 실제 워크로드 균형을 체감하기 어려움.
- **니즈**: 팀원별 할당 작업 수·지연 건수·협업 빈도를 한 화면에서 비교. 이상 신호 자동 알림.
- **시나리오**: 월요일 09:00 출근 직후 대시보드 열기 → 과부하 경고 팀원 확인 → 해당 티켓 리스트 열어 1건 재할당 → 팀챗으로 1:1 요청.

### 5.2 페르소나 B — 팀원 "이지혜" (29세, 백엔드 엔지니어)

- **소속**: SaaS 스타트업, 결제 플랫폼 스쿼드 소속
- **페인포인트**
  - 본인의 기여가 정성적으로만 평가되어 공정성 의문.
  - 다른 팀원에 비해 업무량이 많은지 적은지 비교할 객관적 수치가 없음.
  - 1:1 미팅에서 리더가 보는 숫자와 본인 인식이 달라 대화가 겉돎.
- **니즈**: 본인 성과 요약(완료 티켓, 리뷰 참여, 일정 준수율)을 개인 뷰로 확인. 팀 내 익명 벤치마크 비교.
- **시나리오**: 금요일 오후 개인 탭에서 주간 기여도 리뷰 → 일정 준수율 92% 확인 → 1:1 템플릿에 자동 첨부.

## 6. 사용자 시나리오 / User Flow (3개)

### 6.1 Flow 1: 리더의 주간 팀 점검 (핵심 플로우)

1. 리더가 좌측 내비게이션에서 "팀 성과" 클릭.
2. 기본 탭 "이번 주" 로드 — 팀원별 부하 히트맵, 지연 건수, 협업 빈도 위젯 노출.
3. 히트맵에서 적색(과부하) 팀원 카드 클릭.
4. 해당 팀원의 할당 티켓·최근 7일 활동 타임라인이 우측 패널에 열림.
5. 리더가 티켓 1건을 선택, "재할당" CTA 클릭 → 후보 팀원(저부하 상태) 자동 추천.
6. 재할당 확정 → 해당 팀원/리더에게 인앱 + 팀챗 알림 발송.

### 6.2 Flow 2: 팀원의 개인 기여 리뷰

1. 팀원이 상단 프로필 메뉴 → "내 성과" 진입.
2. 주간/월간 토글 선택, 완료 티켓·평균 리드 타임·PR 리뷰 참여율 카드 로드.
3. "팀 평균 대비 보기" 토글 활성화(익명 처리된 팀 평균선 표시).
4. "1:1 노트로 내보내기" 클릭 → 플로우보드 1:1 템플릿에 자동 삽입.
5. 리더와 공유할지 선택(Private / Share with manager).

### 6.3 Flow 3: 임원 리포트 자동 생성

1. 리더가 대시보드 우상단 "리포트 생성" 클릭.
2. 기간(주/월/분기) · 포함 위젯 선택(체크박스 6종).
3. "자동 요약 생성" — LLM 기반 3단락 인사이트 초안 생성(Editable).
4. PDF/PPTX/도큐스페이스 중 내보낼 포맷 선택.
5. 승인 워크플로 옵션 활성화 시 상위 조직장에게 리뷰 요청 발송.

## 8. 비기능 요구사항

- **성능**: 대시보드 초기 로드 P95 ≤ 2.5초(팀 규모 100명 기준). 위젯 새로고침 P95 ≤ 800ms. 실시간 업데이트 지연 ≤ 5초.
- **확장성**: 단일 팀 최대 500명, 단일 워크스페이스 최대 5,000명까지 수평 확장.
- **보안**: SOC 2 Type II, ISO 27001, GDPR/K-ISMS 준수. RBAC 기반 뷰/위젯별 권한 분리. 개인 데이터는 팀 리더 이상만 조회 가능.
- **접근성**: WCAG 2.2 AA 준수. 색각 이상 대응 팔레트 토글, 키보드 내비게이션 100% 지원, 스크린리더 라벨링.
- **국제화**: 한/영/일/독/불/스 6개 언어. 날짜·숫자·통화 로케일 포매팅, RTL 대비 레이아웃.
- **가용성**: 월 99.95% SLA, 다중 리전(AP-Northeast, US-East, EU-Central) 액티브-액티브.
- **감사 로그**: 위젯 조회·재할당 액션은 365일 보관, 관리자 감사 뷰 제공.

## 9. 디자인 가이드

### 9.1 주요 화면 설명

1. **팀 개요 화면(Team Overview)**: 상단 요약 KPI 4개(평균 일정 준수율, 과부하 인원 수, 협업 건수, 리스크 스코어), 좌측 팀원 부하 히트맵, 우측 주간 추세 차트. 화면 분할 3:7.
2. **팀원 상세 드로어(Member Detail Drawer)**: 우측에서 슬라이드 인. 상단 프로필·이번 주 부하 바, 중단 티켓 리스트(상태·기한·우선순위), 하단 최근 7일 활동 타임라인.
3. **개인 성과 뷰(My Performance)**: 카드 레이아웃 2×3. 완료 티켓/리드 타임/리뷰 참여율/일정 준수율/연속 집중 시간/이번 주 하이라이트 카드. 하단에 "1:1 노트로 내보내기" 고정 CTA.

### 9.2 디자인 원칙

- **Glanceable**: 리더가 10초 안에 팀 상태를 파악하도록 요약 우선 배치.
- **Non-surveillance**: 개인 감시 인상을 피하기 위해 팀원 뷰는 익명 비교 기본값, 세부 추적은 권한 기반 공개.
- **Actionable**: 모든 위젯에 "다음 액션" CTA 포함(재할당, 1:1 예약, 알림 설정).
- **Consistent**: 플로우보드 디자인 시스템 v4 (FlowDS v4) 컴포넌트 100% 재사용.

## 10. 기술 고려사항

- **데이터 소스**: 내부 이벤트 스트림(`task.*`, `comment.*`, `review.*`), 사용자 프로필(관계형 DB `users`), 프로젝트 메타(관계형 DB `projects`), 외부 캘린더 API 연동.
- **실시간성**: 신규 이벤트는 CDC 도구 → 이벤트 스트림 → 스트리밍 집계 엔진 → OLAP 분석 DB에 적재. UI는 WebSocket으로 5초 이내 반영.
- **집계 파이프라인**: 1분/5분/1시간/1일 프리-애그리게이션 테이블 유지. 대시보드 쿼리 P95 ≤ 300ms.
- **LLM 활용**: 리포트 요약은 내부 Gateway(Vendor-agnostic)를 통해 호출. 고객 데이터는 학습에 사용되지 않음(opt-out enforced).
- **데이터 프라이버시**
  - 개인 성과 데이터는 워크스페이스 내에서만 유통, 타 워크스페이스 격리 보장.
  - 팀원 뷰에서 "팀 평균 비교"는 최소 5인 이상일 때만 노출(k-익명성).
  - 유럽/한국 리전 데이터 레지던시 옵션 제공.
  - 팀원은 본인 데이터 다운로드·삭제 요청 가능(GDPR Art. 15/17).
- **신뢰성**: 집계 재계산 배치(nightly) + 실시간 스트림 이중 체계. 드리프트 모니터링 대시보드 운영.

## 14. 부록

### 14.1 용어 정의

- **일정 준수율(On-time Rate)**: 완료된 티켓 중 기한 내 완료된 비율. `(기한 내 완료 / 총 완료) × 100`.
- **부하 스코어(Load Score)**: Open 티켓 수 × 평균 복잡도 가중치 + 마감 임박(≤72h) 티켓 × 1.5.
- **리스크 스코어(Risk Score)**: 부하 스코어, 야간 활동 비율, 연속 마감 임박 횟수, 휴가 미사용 일수를 가중 합산(0~100 스케일).
- **k-익명성**: 비교 그룹 내 개인이 최소 k명(본 제품 k=5) 이상일 때만 집계 결과 공개.
- **GA**: General Availability, 전 고객 대상 정식 출시.
